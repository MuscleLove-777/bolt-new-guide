Bolt.newを始めるための準備と初期設定

「Bolt.new 使い方」をマスターするための第一歩は、環境の準備です。Bolt.newは完全にブラウザ上で動作するため、特別なソフトウェアのインストールは一切不要です。インターネット接続があれば、どこからでもすぐに開発を始められます。

1. Bolt.new公式サイトへのアクセス

  • お使いのWebブラウザで https://bolt.new/ にアクセスします。これがあなたの開発の司令塔となります。

2. アカウントの作成とログイン

  • サイトにアクセスすると、GitHubアカウントやGoogleアカウントを使ったログイン、またはメールアドレスでの登録が求められます。通常、GitHub連携が推奨されますが、任意の連携方法でサインアップを進めてください。
  • 登録が完了すると、Bolt.newのメインインターフェースが表示されます。

3. メインUIの概要理解

Bolt.newの画面は、直感的で分かりやすく設計されています。主に以下のエリアで構成されています。 * プロンプト入力欄: 画面上部または中央に大きく表示される、AIに指示を出すためのテキストボックスです。ここに作りたいアプリの要件を自然言語で入力します。 * プロジェクトパネル: 画面左側に表示され、生成されたファイルの構造やナビゲーションを行います。 * コードエディタ: 画面中央に配置され、生成されたコードを直接編集できます。VS Codeライクな高機能エディタが統合されています。 * プレビューウィンドウ: 画面右側に表示され、開発中のアプリの動作をリアルタイムで確認できます。コードを編集すると、自動的にプレビューが更新されるため、開発効率が飛躍的に向上します。 * ターミナル/コンソール: 下部に配置され、ログの確認やコマンド実行が可能です。

プロンプトでフルスタックアプリを自動生成する手順

Bolt.newの真骨頂は、より複雑なフルスタックアプリケーションもプロンプト一つで生成できる点です。ここでは、Next.js 14とSupabaseを連携させたToDoアプリを例に、具体的な生成手順を解説します。

1. 要件定義のプロンプト作成のコツ

AIに意図した通りのアプリを生成させるためには、具体的で明確なプロンプトを作成することが重要です。以下の要素を含めると効果的です。 * アプリの目的と主な機能: 何をするアプリか、どのような機能が必要か。 * 技術スタックの指定: Next.js (バージョンも), React, TypeScript, Tailwind CSS, Supabase, Prismaなど、使いたい技術を具体的に。 * データベーススキーマ: どのようなテーブルとカラムが必要か（例: todosテーブルにid, task, is_complete, user_idなど）。 * 認証の有無: ユーザー登録、ログイン機能が必要か。 * デザイン要件: モダン、ミニマリスト、特定のコンポーネントライブラリの使用など。

2. プロンプト入力とアプリ生成

Supabase連携のToDoアプリを生成するプロンプトは以下のようになります。

  1. プロンプトを入力: プロンプト入力欄に、以下の指示を入力します。 text Create a full-stack ToDo list application using Next.js 14 with the App Router, TypeScript, Tailwind CSS, and Supabase for authentication and database. Users should be able to sign up, log in, add new tasks, mark tasks as complete, and delete tasks. Tasks should be stored in a 'todos' table in Supabase, with columns: 'id' (UUID), 'user_id' (UUID, foreign key to auth.users), 'task' (text), 'is_complete' (boolean), and 'created_at' (timestamp with default now()). Implement real-time updates for task changes. The UI should be clean and responsive.
  2. 「Generate App」をクリック: ボタンをクリックし、AIがコードを生成するのを待ちます。この際、Next.jsのセットアップ、Supabaseのクライアント設定、基本的なAPIルート、UIコンポーネントなどが自動で生成されます。

3. Supabaseプロジェクトとの連携

AIが生成したコードには、Supabaseとの接続設定が含まれていますが、実際にあなたのSupabaseプロジェクトと連携させるには、環境変数の設定が必要です。

  1. Supabaseプロジェクトの準備:
    • Supabaseの公式サイトにアクセスし、新しいプロジェクトを作成します。
    • プロジェクトダッシュボードで、「Settings」→「API」に移動し、「Project URL」と「Anon Public Key」を控えておきます。
    • 「Database」→「Table Editor」で、AIに指示した通りのtodosテーブルを作成します。権限設定（RLS: Row Level Security）も適切に設定し、user_idに基づいて行へのアクセスを制限できるようにします。
  2. Bolt.newでの環境変数設定:
    • Bolt.newのエディタで、.envファイル（または環境変数設定画面）を開きます。
    • 以下の環境変数を追加します。 text NEXT_PUBLIC_SUPABASE_URL=（あなたのSupabase Project URL） NEXT_PUBLIC_SUPABASE_ANON_KEY=（あなたのSupabase Anon Public Key）
    • これらの変数がコード内で利用されることで、Bolt.newで生成されたアプリがあなたのSupabaseプロジェクトと通信できるようになります。

4. アプリの確認と改善

環境変数を設定し、アプリが正しくSupabaseと連携できれば、プレビューウィンドウでToDoアプリが動作するはずです。ユーザー登録、ログイン、タスクの追加、完了、削除など、全ての機能が想定通りに動くか確認します。

  • プロンプトによる改善: もしUIが気に入らない場合や、追加したい機能がある場合は、再度プロンプト入力欄に指示を追加します。 text Add a dark mode toggle to the ToDo app. text Refine the styling of the task list to use a card-like layout with shadows. このように、対話形式でアプリを洗練させていくことができます。

5. アプリのデプロイ

Bolt.newの大きな強みの一つは、生成したアプリをブラウザから直接デプロイできる点です。

  1. 「Deploy」ボタンをクリック: Bolt.newのエディタ上部にある「Deploy」ボタンをクリックします。
  2. デプロイ先を選択: VercelやNetlifyなど、主要なホスティングサービスとの連携が可能です。GitHubアカウントでログインしている場合、対応するサービスへの連携がスムーズです。
  3. 環境変数の設定（デプロイ先）: デプロイ先サービス（例: Vercel）でも、Supabase連携に必要な環境変数（NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY）を設定する必要があります。これはデプロイサービスのダッシュボードで行います。
  4. デプロイ完了: 数分でアプリがビルドされ、公開URLが提供されます。これで、あなたのフルスタックアプリがインターネット上に公開されました。
